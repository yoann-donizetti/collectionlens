# CollectionLens — Vision Produit

## 1. Présentation du projet

CollectionLens est une plateforme intelligente de découverte culturelle centrée sur l’exploitation des données, les systèmes de recommandation et l’intégration progressive de fonctionnalités d’intelligence artificielle.

Le projet vise à proposer une expérience personnalisée permettant aux utilisateurs de :

• Gérer leurs collections culturelles
• Découvrir de nouveaux contenus adaptés à leurs goûts
• Explorer leurs préférences culturelles
• Exploiter intelligemment les métadonnées des œuvres

Le périmètre initial du projet se concentre sur :

• Les mangas
• Les comics
• Les bandes dessinées

L’architecture est cependant pensée dans une logique évolutive afin de permettre l’intégration future d’autres univers culturels, notamment les jeux vidéo.

---

## 2. Problématique

Les collectionneurs et passionnés de culture utilisent aujourd’hui de nombreux outils fragmentés pour :

• Gérer leurs collections
• Rechercher des œuvres
• Découvrir de nouveaux contenus
• Suivre leurs lectures

Les solutions existantes présentent plusieurs limites :

• Manque de centralisation
• Faible personnalisation
• Recommandations limitées
• Exploitation réduite des données utilisateur
• Peu d’intégration de technologies IA modernes

CollectionLens cherche à répondre à ces problématiques en proposant une plateforme orientée donnée, recommandation et découverte culturelle.

---

## 3. Objectifs du projet

Les principaux objectifs de CollectionLens sont :

• Centraliser les collections utilisateur
• Enrichir automatiquement les œuvres via des APIs externes
• Structurer et exploiter les métadonnées culturelles
• Construire des mécanismes de recommandation personnalisée
• Analyser les similarités entre œuvres
• Développer progressivement des fonctionnalités conversationnelles basées sur l’IA

Le projet vise également à servir de support d’expérimentation autour :

• De la Data Science
• Des systèmes de recommandation
• Du NLP
• Des embeddings vectoriels
• Des architectures RAG

---

## 4. Vision produit

CollectionLens ne cherche pas uniquement à être une application de gestion de collection.

Le projet vise à construire une plateforme capable d’exploiter intelligemment les données culturelles afin de proposer :

• Une découverte personnalisée de contenus
• Des recommandations contextualisées
• Une exploration intelligente des œuvres
• Une expérience utilisateur enrichie par l’IA

L’objectif à long terme est de transformer CollectionLens en une plateforme hybride combinant :

• Gestion de collection
• Analyse de données culturelles
• Recommandation intelligente
• Recherche conversationnelle
• Personnalisation avancée

---

## 5. Périmètre du projet

### Phase POC

Le POC a pour objectif de valider :

• La récupération des métadonnées via APIs externes
• La structuration des œuvres, séries et volumes
• La qualité des données récupérées
• La gestion d’une collection utilisateur
• La faisabilité d’un premier système de recommandation

La recommandation reposera principalement sur une approche par contenu exploitant :

• Les genres
• Les tags
• Les descriptions
• Les synopsis
• Les similarités entre œuvres

Les modèles de langage pourront être utilisés en complément afin de contextualiser et reformuler les recommandations proposées.

### Phase MVP

Le MVP visera à proposer une première version exploitable de la plateforme intégrant :

• Gestion de collection stabilisée
• Recommandation améliorée
• Analyse des préférences utilisateur
• Dashboard et statistiques
• Premiers mécanismes IA

### Évolutions futures

Les évolutions futures pourront inclure :

• Embeddings vectoriels
• Similarité sémantique
• Recherche conversationnelle
• Architectures RAG
• Fonctionnalités communautaires
• Extension vers les jeux vidéo

---

## 6. Positionnement Data / IA

CollectionLens s’inscrit dans une démarche de conception de solution Data/IA complète mêlant :

• Gestion et structuration des données
• Exploitation de métadonnées culturelles
• Systèmes de recommandation
• NLP et recherche sémantique
• Intelligence artificielle conversationnelle
• Analyse utilisateur et personnalisation

Le projet adopte une approche progressive permettant de construire des fonctionnalités IA de manière réaliste et évolutive.

## 7. Stack technique envisagée

Les technologies envisagées à ce stade incluent notamment :

| Domaine         | Technologies envisagées                         |
| --------------- | ----------------------------------------------- |
| Backend         | Python                                          |
| Data Science    | Pandas, Scikit-learn                            |
| APIs            | Google Books API, OpenLibrary, APIs culturelles |
| Recommandation  | Similarité contenu, NLP                         |
| IA              | Embeddings, LLM                                 |
| Base de données | SQLite / PostgreSQL                             |
| Interface       | Streamlit                                       |
| Versioning      | Git / GitHub                                    |

Cette stack pourra évoluer progressivement selon les besoins du projet.
